Introdução

1.1 Quais são os 3 tipos de códigos básicos de uma página de hipertexto? Explique para que serve cada um dos tipos.

HTML - HyperText Markup Language, traduzindo, Linguagem de Marcação de Hipertexto, ou seja, é uma linguagem de marcação. Em síntese, ela é a responsável por definir quais elementos serão exibidos em uma determinada página

CSS - Cascading Style Sheets, ou folhas de estilo em cascata, portanto, é uma linguagem de folhas de estilo, ou uma linguagem de definição de estilos. Ela é responsável por definir como os elementos de conteúdo serão mostrados.

JavaScript - é uma linguagem de programação. Ela pode ter diferentes propósitos em uma página web e, pelo fato de ser uma linguagem de programação, requer conhecimentos de lógica de programação


1.2 Quantos arquivos CSS e Javascript podem estar vinculados a uma mesma página de hipertexto? Justifique sua resposta.

Uma página de hipertexto pode estar vinculada a inumeros arquivos de CSS e JavaScrip, por exemplo, uma página pode ter um arquivo CSS para estilizar o layout, e outro para estilizar os botões.


1.3 É possível diferentes páginas de hipertexto terem vínculo com os mesmos arquivos Javascript e CSS? Justifique sua resposta.

Sim, um arquivo CSS pode ser compartilhado por diferentes páginas de hipertexto, ao mudar a típografia no arquivo designado por exemplo as diferentes páginas que compoem o site mudaram de acordo, facilitando a manutenção, quanto ao JavaScrip,também é comum que as páginas compartilhem os mesmos códigos. É possível ter um código para mostrar uma janela com mensagem, para animar um elemento da página, para atualizar apenas uma parte da página, entre outros.


1.4 Descreva a diferença entre uma página de hipertexto estática e uma dinâmica, pontuando as vantagens e desvantagens de cada um dos dois tipos.

Uma página de hipertexto estática é composta de arquivos prontos, que simplesmente serão baixados do servidor web que hospeda o site. Vantagens: Facilidade de manutenção para conteúdo pouco alterado. Desvantagens: Dificuldade em casos de atualização frequente.

Já uma página de hipertexto dinâmica é gerada por um programa especial chamado de aplicação web, que roda no servidor web e que gera o código-fonte da página sempre que ela é solicitada por um usuário. Vantagens: Atualização dinâmica de conteúdo. Desvantagens: Maior carga de processamento no servidor.


1.7 Quais as vantagens da atualização parcial de conteúdo em segundo plano em uma página de hipertexto usando Javascrip?

Carregar conteúdo do servidor web em segundo plano, permite atualizar partes da página sem a necessidade de requisitar toda a página novamente, por exemplo, a atualização do placar de jogos de futebol em tempo real. Em vez de recarregar toda a página a cada atualização do placar, o Javascript é utilizado para fazer uma consulta ao servidor web em segundo plano, obtendo apenas os dados atualizados do placar. Em seguida, o Javascript atualiza dinamicamente apenas a parte da página que exibe o placar, sem afetar o restante do conteúdo. Essa abordagem oferece vantagens significativas, como uma experiência de usuário mais rápida e fluida, uma vez que apenas as partes que mudaram precisam ser atualizadas, evitando atrasos causados por recarregamentos completos da página. Além disso, reduz a carga no servidor, pois as solicitações de atualização são menores e mais rápidas, já que não é necessário reenviar todo o conteúdo da página a cada atualização.
